## References

### Online
- [OWASP Official](https://owasp.org): The official OWASP portal for application security projects, standards, and community resources.
- [OWASP Top 10](https://owasp.org/www-project-top-ten/): A widely used list of the most critical web application security risks.
- [Microsoft STRIDE](https://learn.microsoft.com/en-us/azure/security/develop/threat-modeling-tool-threats): A threat modeling method covering Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, and Elevation of Privilege.
- [NIST Cybersecurity Framework](https://www.nist.gov/cyberframework): A risk-based framework for managing cybersecurity through Identify, Protect, Detect, Respond, and Recover functions.
- [CIS Critical Security Controls v8](https://www.cisecurity.org/controls/cis-critical-security-controls-list): A prioritized set of actionable safeguards to reduce common cyber risks.
- [MITRE ATT&CK Framework](https://attack.mitre.org): A knowledge base of adversary tactics and techniques based on real-world observations.
- [OWASP ASVS](https://owasp.org/www-project-application-security-verification-standard/): A verification standard with security requirements for designing, building, and testing secure applications.
- [AWS Well-Architected Security Pillar](https://docs.aws.amazon.com/wellarchitected/latest/security-pillar/welcome.html): Security best practices for designing and operating workloads on AWS.
- [Google Cloud Security Foundations Guide](https://cloud.google.com/architecture/security-foundations): Guidance for building a secure baseline architecture and governance model on Google Cloud.
