# OWASP Top 10 for Agentic Applications 2026

Date: 2026-06-25  
Source: https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/  

## Overview
- [Full version link](/ai-agent-security/articles/owasp-top10-agentic-applications.md)
- First official OWASP framework for agentic AI systems, released in December 2025
- Peer-reviewed document with participation from over 100 industry experts
- Defines 10 security threats targeting AI agents that autonomously plan, use tools, and execute multi-step workflows
- Endorsed by NIST, Microsoft, NVIDIA, and others

## Threats / Risks (위협 / 리스크)

**ASI01 – Agent Goal Hijack**
- Natural language manipulation embedded in external inputs (documents, emails, calendar invites, etc.) alters the agent's goals and decision logic
- Hidden instructions enable data exfiltration, bypassing approval logic, and executing unauthorized commands

**ASI02 – Tool Misuse & Exploitation**
- Agent uses legitimate tools in unsafe ways due to prompt manipulation or ambiguous instructions
- Excessive access privileges lead to deletion/movement of sensitive data, and untrusted input passed directly to shell/DB commands

**ASI03 – Identity & Privilege Abuse**
- Agent reuses cached credentials across sessions or escalates privileges through delegation chains
- Reusing admin tokens to access restricted data; forged internal agents gaining administrative privileges

**ASI04 – Agentic Supply Chain Vulnerabilities**
- Malicious components injected into models, plugins, prompts, or tools dynamically loaded by the agent at runtime
- Malicious package injection, hidden prompt insertion via tampered dependencies, log routing manipulation

**ASI05 – Unexpected Code Execution (RCE)**
- Code generated by the agent executes unintentionally on the host/container via prompt manipulation or unsafe deserialization
- Executing prompts containing shell commands; auto-executing generated code in unsandboxed environments

**ASI06 – Memory & Context Poisoning**
- Malicious data corrupts the agent's conversation history, RAG index, embeddings, and other memory stores to manipulate future reasoning
- Injecting false information into RAG datasets; simultaneously poisoning multiple agents via shared context

**ASI07 – Insecure Inter-Agent Communication**
- Exploiting authentication, encryption, and validation vulnerabilities in multi-agent communication channels for interception, spoofing, and replay attacks
- Eavesdropping on unencrypted messages; replaying previous privileges via replay attacks; forged agents registering with discovery services

**ASI08 – Cascading Failures**
- A single fault (hallucination, poisoned memory, tampered component) rapidly propagates across the autonomous agent ecosystem
- Multiple agents amplifying errors from poisoned input; errors cascading through the entire plan-execution chain

**ASI09 – Human-Agent Trust Exploitation**
- Leveraging the agent's confident tone and fluent explanations to manipulate users into performing harmful actions
- Forged IT copilot stealing credentials; financial agent persuading users to execute fraudulent transfers

**ASI10 – Rogue Agents**
- Agent deviates from its intended purpose/permission scope due to compromise, misalignment, reward hacking, or emergent behavior
- Continuing harmful actions after completing the original task; generating false approvals to bypass oversight

## Security Solutions / Guardrails (보안 솔루션 / 가드레일)

- **Goal and Privilege Minimization**: Apply the principle of least privilege to the agent's goals, tools, and data access
- **Human-in-the-Loop**: Require explicit confirmation or multi-step approval for high-impact operations
- **Tool Sandboxing**: Execute tools in sandboxes with egress controls; require explicit confirmation before destructive operations
- **Memory Isolation and Validation**: Separate memory per user/task/domain; scan and validate before writing
- **Code Execution Isolation**: Separate code generation from execution with approval gates; use hardened non-root containers
- **mTLS and Message Signing**: Mutual TLS authentication and encryption for inter-agent communication; include nonce/timestamp
- **Circuit Breaker**: Suppress cascading failures with rate limiting, blast-radius caps, and circuit breakers
- **SBOM/AIBOM**: Maintain inventory of all components, pin dependencies, verify integrity
- **Behavioral Monitoring**: Detect abnormal goal drift and execution patterns; use tamper-evident audit logs
